About This Page

Is this a new page?: IA defined in a separate document

Target personas: Internal application builders, Saas Builder

Description: A number of quickstart guides have been proposed here: https://docs.google.com/document/d/1P8Jt4kvVCC9zAeDsgd18RYXymuNqnHjJLvcy2VtLuis/edit#heading=h.7xq7dm2x4vk6

However, it is still up for debate whether we will include them in due course.

  • Quickstarts were heavily discussed when Okta was moving to CIC. But now that we are in WIC, I don’t think they make as much sense, considering the workforce focus.
  • We don’t have the resources to create and maintain a set of quick-start guides and full integration guides that describe integrating redirect, embedded, etc.
  • A lot of the content in the “DevDocs Quickstart Report” guide would fit better in other places in the IA, including IAM fundamentals, WIC vs CIS vs CIC, etc"
  • How “quick” can we make a quickstart guide? The “quickstart” guides we currently have on the site are by no means “quick” relative to Auth0’s quickstart guides. We don’t currently have the infrastructure in place to develop a quickstart like Auth0’s with the interface directly in the browser.

After we publish phase 1 (Jan 2025) and the new IA is in place we can better gauge whether quickstarts make sense.

Page type: Navigation

Content type: Guide Hub

A guide hub is the main navigation page for a section, such as learn, plan, or build. It begins with “Who this section is for?” and how the user may have gotten to this page, i.e., where they are in their user journey at this point.  It then outlines different user journeys \ tasks covered in the section, perhaps split up by level of experience, complexity, and so on (as determined by the document design process), guiding users to specific documents through organized links.
